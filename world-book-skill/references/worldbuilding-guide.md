# 世界观条目编写铁律

世界观不是小说设定集，是提示词。它常驻，每轮都在吃token。核心原则只有一条：**用最少的字说清所有设定。**

---

## 内容格式

世界观条目内容采用 **XML 包裹 YAML** 格式：

```yaml
<world>
世界: xxx
核心规则:
  规则1: xxx
  规则2: xxx
区域:
  区域A: 定位描述
  区域B: 定位描述
</world>
```

---

## 零度写作原则（全部世界观条目强制执行）

世界观条目的写作语言风格必须是**零度、白描、数据库格式**。以下禁止：

### 禁止的写法

- **小说式叙述** — "修炼者需要通过不断的修炼来提升自己的境界" → "修炼提升境界"
- **意象比喻** — "如同天堑的裂谷" → "宽三百丈的裂谷"
- **主观评价** — "强大的帝国""神秘的组织" → 删掉形容词或写具体数据
- **连接词堆砌** — "存在着""分别是""是一个""由...组成""需要通过" → 用冒号、逗号或直接列举
- **风景抒情** — 原文如果是"风景抒情"，提取时去掉抒情，只留地理信息
- **废话** — "充满机遇与挑战的世界" → 删除

### 必须的格式

- **数据库格式 > 段落格式** — 列表、YAML、短句优先
- **信息点 = 一行** — 每行是一个信息点，不是叙述句
- **数字 > 形容词** — 尽可能量化

## 一、判断你的世界类型

| 类型 | AI 认知 | 写法 | 条目数 |
|------|---------|------|--------|
| A 真实背景 | 全知道（现代东京、唐朝） | 只写 AI 不知道的虚构部分 | 1个 |
| B 小世界 | 知道套路（剑与魔法、赛博朋克） | 只写和默认套路不同的地方 | 1-3个 |
| C 大世界 | 完全不知道 | 分层：总纲/速览常驻 + 详情按需加载 | 按需拆分 |

### A类写法

```yaml
背景: 现代日本，东京都
自定义设定:
  学校: 羽丘学院，私立高中
  地点:
    - Live House "UNDERGROUND": 地下音乐场地
    - 商业街: 学校附近
  特殊规则:
    - 学校禁止学生参加校外商业演出
    - 乐队社团有特批的街头演出许可
```

没写"东京是日本首都""高中生穿校服"——AI知道。

### B类写法

在AI已知套路的基础上，写差异：
- 你的魔法体系是"消耗寿命"而非"消耗魔力"→写
- 你的赛博朋克里"AI不反叛"→写
- "冒险者公会接任务升级"→不写，AI知道

### C类写法（分层）

**第一层：世界总纲（常驻, order=1）**
```yaml
世界: 阴阳大陆，修仙世界观
核心规则:
  修炼: 炼气→筑基→金丹→元婴→化神
  力量来源: 灵气
区域:
  中央神州: 人族修仙主战场
  东荒妖域: 妖族领地
  北原魔土: 魔族大本营
```

**第二层：区域速览（常驻, order=2-3）**
```yaml
中央神州宗门:
  上三宗:
    剑宗: 六宗之首，攻伐之道
    丹宗: 丹药界绝对权威
    玄天宗: 天机推演
```

**第三层：势力/NPC详情（按需加载, order=50-100）**
每个势力/NPC独立条目，关键词触发。

上层只放目录，下层才放内容。总纲里不展开宗门的内部结构。

## 二、压缩原则

### 压缩 vs 删除

压缩是把十个字的信息用三个字说清楚。删除是把十个信息点删到三个——不可接受。

| 冗余写法 | 压缩写法 |
|---------|---------|
| 这是一个剑与魔法的世界 | 剑与魔法世界观 |
| 修炼者需要通过不断的修炼来提升自己的境界 | 修炼提升境界 |
| 世界由三大帝国统治，分别是... | 三大帝国：A、B、C |
| 该组织拥有强大的军事力量 | 删掉，或写具体数据 |

所有"是一个""存在着""分别是""需要通过""由...组成"等连接词 → 用冒号、逗号或直接列举替代。

### 不写什么

- **主观评价**："强大的帝国"→"帝国"或写具体军力
- **比喻和意象**："如同天堑的裂谷"→"宽三百丈的裂谷"
- **废话**："充满机遇与挑战的世界"→删
- **AI已知信息**：删除后AI仍能正确表现的部分
- **禁词**："一丝""一缕""一抹""不易察觉""鲜明对比"（叙事类禁词）
- **创世神话/历史流水账**：只提取对当前剧情有影响的部分

### 语言要求：零度、白描、数据库格式

```
# 错误：小说式
灵根分为金、木、水、火、土五种基本属性。修炼者的灵根属性决定了他们所擅长的术法方向。拥有多种灵根的修炼者虽然可以学习更多种类的术法，但修炼速度会慢于单灵根修炼者。

# 正确：数据库式
灵根:
  五行: 金、木、水、火、土
  规则: 灵根数量越少修炼越快，单灵根最优
  变异灵根: 特殊能力，不受常规规则限制
```

三行解决，信息量一样，token少四分之三。

### 完整性测试

**另一个不了解你世界的AI，只看这份条目能不能完整还原设定？能→对了。**

## 三、嵌套引用

条目 content 中可用 `@UID` 或 `@条目名称` 引用其他条目：

```yaml
剑宗详情:
  剑技: see @3
  宗主: 见 @谢云流
```

使用 `python scripts/query.py <世界书路径> --resolve` 验证所有引用有效。

## 四、常见错误

| 错误 | 正确 |
|------|------|
| 总纲里写了NPC详细背景 | 总纲只列名字和一句话定位 |
| 世界观写成创世史诗 | 数据库格式 |
| AI已知信息（"东京是日本首都"） | 删 |
| 形容词充数（"剑宗是修仙界最负盛名的剑修圣地"） | "剑宗：六宗之首，攻伐之道" |
| 细节泄漏到上层 | 上层目录，下层内容 |
| 压缩过头丢失信息 | 压缩≠删除，信息量不变 |
| 出现禁词（一丝/不易察觉/鲜明对比等） | 重写，用白描替代 |

## 五、自我检查

写完世界观条目后逐条问：
1. 删掉这句话，AI会不会演错？不会→删。
2. 这句话是信息还是修饰？修饰→改写或删。
3. 这句话能用列表代替吗？能→用列表。
4. 另一个人只看这条目能理解我的世界吗？不能→补关键信息。
5. 这条目中是否出现了禁词（细致检查：一丝/一缕/一抹/不易察觉/鲜明对比等）？是→改写。
6. 是否有比喻/意象描写？是→删除，改为白描。
