## 所有条目必须遵守的铁律

以下规则适用于**所有任务类型的所有条目**：

### 配置铁律

1. **双递归** — 所有条目 `--prevent-recursion --exclude-recursion`
2. **蓝绿灯正确** — 单角色卡全蓝 / 多角色卡速览蓝+详情绿 / NPC/场景/故事章节全绿
3. **位置正确** — 世界观↑Char / 角色↓Char / 文风↑AT / 禁词↑AT / 行为纠正@D-0
4. **关键词英文逗号** — 中文逗号和空格直接失效

### 内容格式铁律

5. **所有世界书条目内容采用 XML 包裹 YAML 格式**：
   ```yaml
   <tag>
   key1: value1
   key2:
     - item1
     - item2
   </tag>
   ```
   而不是纯XML格式（`<tag>key:value</tag>`）

### 内容铁律（零度写作原则）

6. **不写意象比喻** — "眸如星辰""指节发白""心弦拨动"等全部删除
7. **不写解释性修饰** — 不对角色动作/神态进行作者视角的解释（"这个动作体现了他的紧张"）
8. **不写不存在的事物** — "拂去不存在的灰尘""推不存在的眼镜"
9. **不写主观评价** — 删除"温柔""强大""精致"等形容词，转化为行为描写或数据
10. **不写AI已知信息** — 删除后AI仍能正确表现的内容
11. **数据库格式 > 散文格式** — 列表、YAML、短句优于段落长文

### 禁词条目强制创建

12. **所有世界书必须包含 3 条禁词条目**（position=2, 蓝灯, constant）：
    - 叙事禁词：一丝/不易察觉/鲜明对比/喉结/弧度/先否认再肯定句式等
    - 比喻禁词：石子/湖面/拉满的弓等比喻、指节发白、睫毛特写等
    - 描写禁律：禁止解释性修饰、禁止描写不存在事物、禁止对白精确数字

### MVU 和 HTML 的铁律

13. **MVU ZOD / 变量系统 — 仅用户明确要求时才启用**
14. **HTML 美化 / 状态栏 — 仅用户明确要求时才启用**
15. **不主动建议，不主动添加。** 你没有说出口的建议才算合格。

详见 `config-guide.md` 第四节和 SKILL.md。
